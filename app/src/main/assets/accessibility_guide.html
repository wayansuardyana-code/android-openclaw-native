Accessibility Service Setup

Required after every app update

Why do I need to do this?

Android automatically disables Accessibility Services when an app is updated or reinstalled. This is a security feature built into Android — it cannot be bypassed.

OpenClaw needs Accessibility Service to read your screen, tap buttons, type text, and control apps. Without it, the AI agent cannot interact with your device.

Steps to Re-enable

1 Open Settings on your device

2 Go to Accessibility

On some devices: Settings → Additional Settings → Accessibility

3 Find OpenClaw (or "Installed apps/services")

Samsung: Settings → Accessibility → Installed apps
Xiaomi: Settings → Additional Settings → Accessibility
Pixel: Settings → Accessibility → OpenClaw

4 Toggle ON the switch for OpenClaw

5 Tap "Allow" on the confirmation dialog

Android will warn you about accessibility permissions — this is normal. OpenClaw needs these to control your device on your behalf.

Also Enable: Notification Access

1 Go to Settings → Apps → Special app access → Notification access

Or: Settings → Notifications → Notification access (varies by device)

2 Toggle ON for OpenClaw

Quick check: After enabling, go to OpenClaw → Chat → type /status. If it shows "Service: Running", you're all set!

Troubleshooting

Can't find OpenClaw in Accessibility list?

Force stop OpenClaw → reopen it → check again. Some devices need a restart.

Accessibility keeps turning off?

Some devices have "battery optimization" that kills background services. Go to Settings → Battery → OpenClaw → set to "Unrestricted" or "No restrictions".

Samsung devices:

Disable "Put unused apps to sleep" in Settings → Battery → Background usage limits. Add OpenClaw to "Never sleeping apps".

Xiaomi/MIUI devices:

Settings → Battery → App battery saver → OpenClaw → "No restrictions". Also: Security → Manage apps → OpenClaw → Autostart → ON.
